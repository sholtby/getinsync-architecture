The QuickBooks Principle

QuickBooks doesn't ask you to "create an accounts payable transaction." It shows you a check and you fill it in. The visual is the form. You always know what you're doing because the metaphor tells you.

Design rule: The right panel always shows where the data the user is entering fits in the architecture. The diagram builds itself as they go — empty placeholders become real nodes. By the time they finish, they've built a complete architecture picture without knowing that's what they did.

Architecture Preview

Two Forms, One Wizard

The user self-selects their complexity level. Simple is the default. Advanced is what exists today, accessed via "I need more options" or by choosing the IT Service cost path.

Day 1 Maturity

Simple Form (default)

Business Owner / App Manager

"I know the app name, who owns it, and roughly what we pay."

  • REQ Application Name
  • REQ Workspace
  • REQ Hosting Type (SaaS / On-Prem / Cloud / Hybrid)
  • OPT Description
  • OPT Business Owner
  • OPT Annual Cost (cost bundle)
  • OPT Vendor
  • OPT Contract End Date

Auto-creates: 1 DP (Primary), 1 Cost Bundle if cost entered.
DP name = App name. Zero CSDM jargon exposed.

Day 90 Maturity

Advanced Form (opt-in)

IT Architect / ITSM Lead

"I need to map deployment profiles, IT services, and tech composition."

  • REQ Everything from Simple, plus:
  • OPT Multiple Deployment Profiles
  • OPT Environment / Region / Cloud Provider
  • OPT Server Name / DR Status
  • OPT Technology Products (OS, DB, middleware)
  • OPT Software Products linked
  • OPT IT Service allocations
  • OPT Integrations
  • OPT Tech Debt Notes / Remediation Effort
  • OPT Operations teams

This is essentially today's Edit Application tabs, surfaced as a guided wizard with the visual anchor.

Behavior Rules

Visual Anchor

  • Visible in the right panel during Add/Edit — collapsible via toggle handle on left edge
  • Default open for new users (teaching tool). Collapse state persisted in localStorage so power users stay collapsed
  • Hidden by default on viewports below 900px
  • Ghost placeholders (dashed, gray) show what *could* be filled in — teaches the model without forcing it
  • Nodes light up with real data as fields are completed — the "aha moment"
  • Matches the ArchiMate-informed design system: rounded corners for apps, tight corners for DPs, color = health/status
  • Clickable — user can tap a node to jump to that section of the form

Simple → Advanced Graduation

  • Simple form is always the default for "Add Application"
  • "I need more options" link at bottom of Simple form opens Advanced
  • Choosing "IT Services" cost path auto-switches to Advanced (you can't wire IT Services in Simple mode)
  • Edit Application opens in the mode that matches the app's current complexity — if it has IT Services, open Advanced
  • No data loss switching between modes — Advanced is a superset
  • Integrations are graduated: Simple captures the connection (app, direction, method). Advanced adds data tags, sensitivity, frequency, DP-level scoping, criticality

The 18-Year-Old Test

  • Simple form: no terms like "Deployment Profile," "Cost Bundle," "IT Service," or "CSDM"
  • Simple form labels: "Where does it run?" not "Hosting Type." "What does it cost?" not "Annual Run Rate."
  • Advanced form can use precise terminology — the user opted in

Implementation Notes

Schema impact: None. This is a pure frontend change. The Simple form auto-creates the same underlying records (application + DP + cost bundle) that the current UI creates manually. The visual anchor reads from the same data the Edit tabs already display. No new tables, columns, or RLS policies needed.

Dependencies

  • Demo data reset (COR) should complete first — the visual anchor needs fully wired apps to look compelling
  • React Flow (already in codebase for Visual tab) can power the anchor diagram
  • The anchor is a simplified, non-interactive version of the Visual tab Level 2 layout

Estimated Effort

  • Visual anchor component: M (reuses React Flow + existing design tokens)
  • Simple form wizard: S-M (subset of existing fields, auto-create logic)
  • Wiring anchor to form state: S (React state → node visibility)
  • Total: ~1 sprint if prioritized, or can be split across releases
